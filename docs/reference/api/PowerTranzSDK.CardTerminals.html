Classes

PTZCardTerminal

The card terminal

Interfaces

IPTZCardTerminal

The interface all terminals must adhere to

Delegates

DeviceTextCompletion

Internal delegate used by the SDK

DidConnectTerminal

This delegate function is called when a communication channel has been opened with the credit card terminal.

DidDisconnectTerminal

This delegate function is called when the communication channel to the credit card terminal has been closed.

DidFailToConnectTerminal

This delegate function is called when the connection to the Terminal hardware fails to be created.

DidFailWithError

This delegate function is called when an error has been detected when performing a requested task.

DidFailWithReversal

This method is called when a reversal is performed

DidFinishTransactionWithResponse

This delegate function is called after the terminal has generated the Transaction Certificate or Application Authentication Cryptogram and no more online requests are pending.

DidReceiveCardEvent

This delegate method is called when the terminal receives a card event. It does not return the card data and therefore is called in the event that the card data is not going to be returned for whatever reason.For instance, a card swipe does not return data if the card is IC capable and was swiped.

DidReceiveResponse

This method is used to convey the response body that was sent from the server

DidRequestAccountTypeSelection

This delegate method is called when the terminal is incapable of presenting the account type selection to the card holder. If this method is implemented then the delegate must call back the referencing object or the transaction will timeout.

DidRequestDevicePromptText

This delegate method is called when the terminal would like text displayed by the user. If the terminal is capable of displaying text (terminalCanDisplayText) and this method is implemented, then the terminal should be sent a text string to display to the user. This can be done via the completion handler, with a NIL string to have the terminal display default text.

DidRequestSelectEMVApplication

This delegate method is called when the terminal is not configured to present the application options to the card holder directly. When the hardware supports it, the terminal will manage application selection automatically( Miura for instance).

DidReturnAmountConfirmation

This delegate method is called by some devices when the user confirms/rejects the transaction amount on the terminal hardware Miura devices do not call this method.

OnBatteryLow

This delegate method is called when the terminal sends a low battery status message.

PaymentCompletion

Internal delegate used by the application

PromptCanceled

This delegate function is called when a user presses cancel on the keypad.

SelectDevice

This method is called when a bluetooth connection attempt is made and there may be one or more devices that are available to be paired with.

TerminalDidReceiveKeypadInput

This method is called when a key is pressed on a supported device (Miura). For EMV / PIN security, the key press will not be relayed to the delegate and therefore this message is only useful if the application cares about keypad presses for whatever reason.

TerminalDidRequestFinalConfirmation

This delegate method is called when the terminal is not configured to request final transaction amount confirmation from the card holder directly This is typical on audiojack devices or other devices that have no keypad.

TerminalDidRequestTransactionAmount

This delegate method is called by Anywhere Commerce devices if an EMV transaction is started without an amount set. In general, this should only be used in testing as the EMV spec wants you to start the transaction after the final amount is set.

TerminalPaymentFailed

This delegate function is called when an attempt to make a payment failed with a non-authorization related error.

WillConnectTerminal

This delegate function is called when the communication channel to the credit card terminal could not be opened, but will be reattempted.

WillSendRequest

This method is used to convey the request headers and body to be sent to the server
